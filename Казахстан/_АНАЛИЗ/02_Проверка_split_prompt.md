# Проверка split-prompt: где Staffix переплачивает за кэш

Дата: 2 июля 2026. Проверка по коду, продолжение разбора себестоимости (`01_Модель_себестоимости.md`).

## Вывод одной строкой

**Split-prompt (дешёвый кэш) работает только в Telegram. В каналах Meta — WhatsApp, Instagram, Facebook — его НЕТ: там единый системный промпт, и кэш ломается почти на каждом сообщении.** Приоритетные каналы клиник КЗ (Instagram, WhatsApp) — как раз на дорогом пути.

## Доказательства по коду

**Telegram (оптимизировано):** `src/lib/telegram/ai.ts` + `src/lib/ai-memory.ts` — промпт отдаётся ДВУМЯ блоками:
```
systemBlocks = [
  { text: stable,       cache_control: { ttl: "1h" } },  // бизнес-контекст (услуги/FAQ/база знаний)
  { text: variableTail, cache_control: { ttl: "5m" } },  // клиентский контекст (имя/история/summary)
]
```
Стабильная часть переиспользуется десятки раз (cache_read $0.30/M), меняется только дешёвый хвост.

**Каналы Meta (НЕ оптимизировано):** `src/lib/channel-ai.ts` — промпт единой строкой:
- `L543` `systemPrompt = buildChannelSystemPrompt(biz, channel)` — стабильная часть
- `L553` `systemPrompt += клиентский контекст` — изменяемая часть, дописана в ту же строку
- `L556` `systemPrompt += корректировки`
- `L603` `systemPrompt += "⚠️ база знаний обновлена..."`
- `L644` / `L713` `system: systemPrompt` — одна строка → один cache_control на весь промпт (TTL 1ч)

Итог: любое изменение клиентского хвоста инвалидирует кэш ВСЕГО промпта (включая базу знаний до 50 000 знаков) → `cache_create` заново на каждом сообщении. Это ~$0.064 только на промпт для 17k-токенного клиента → и есть наблюдаемые $0.10.

Подтверждение — комментарий в самом коде (`ai-memory.ts`): «…каждое сообщение стоило ~$0.10 вместо ~$0.005… главная причина дрейфа $11/день». Фикс применили к Telegram, до каналов Meta он не дошёл.

## Что это даёт по деньгам (оценка)

Если перенести split-prompt в `channel-ai.ts`, стабильная бизнес-часть будет читаться из кэша ($0.30/M вместо записи $6/M за 1ч). Ожидаемо: **$0.10 → ~$0.01–0.02 за сообщение** на каналах Meta (порядок как в Telegram). Для клиента уровня Right Flight (1700 сообщений) это **~$170 → ~$20–35/мес**. Оценка порядка, точную даст замер.

## Предлагаемый фикс (код НЕ трогаю без решения — правило Staffix #1)

В `channel-ai.ts` собрать `systemBlocks` как в Telegram: стабильную часть (`buildChannelSystemPrompt`) и изменяемый хвост (клиентский контекст + корректировки + предупреждение об обновлении базы) — двумя блоками с явным `cache_control` (1ч / 5м), и передавать массив в `system` вместо склеенной строки.

Важные детали при внедрении:
- `withPromptCaching` для массива ставит `cache_control` только на ПОСЛЕДНИЙ блок — поэтому cache_control на стабильный блок надо задать ЯВНО (как это сделано в `ai-memory.ts`), иначе станет хуже.
- Порядок блоков: стабильный ПЕРВЫЙ (префикс кэша), изменяемый — второй.
- Проверить, что предупреждение «база знаний обновлена» и корректировки уходят в variable-блок, а не ломают stable.
- Аккуратно с историей сообщений (она в `messages`, не в system — там всё ок).
